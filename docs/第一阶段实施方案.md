# 图数据库Rust实现 - 第一阶段实施方案

## 项目背景

本项目旨在将现有的NebulaGraph图数据库重写为轻量级单节点Rust实现，专注于个人和小规模应用。第一阶段的目标是完成项目设置和核心基础设施的搭建，为后续开发奠定基础。

## 设计目标

1. 创建完整的Rust项目结构
2. 定义核心数据结构
3. 设置基本配置管理
4. 遵循现有graphDB项目结构和代码规范
5. 使用简化版的数据结构，降低复杂性

### 2. 核心数据结构设计

当前graphDB的结构已经基本符合要求，但需要根据Nebula的实现进行优化：

#### 2.1 Node结构（顶点）
```rust
use std::collections::HashMap;
use serde::{Deserialize, Serialize};

/// 表示图中的节点
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
pub struct Node {
    pub id: u64,                    // 节点唯一标识符
    pub labels: Vec<String>,        // 节点标签列表
    pub properties: HashMap<String, Value>,  // 节点属性
}
```

#### 2.2 Edge结构（边）
```rust
/// 表示图中的边
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
pub struct Edge {
    pub id: u64,                    // 边的唯一标识符

    pub to_node: u64,               // 目标节点ID
    pub edge_type: String,          // 边的类型
    pub properties: HashMap<String, Value>,  // 边的属性
}
```

#### 2.3 Value结构（通用值类型）
```rust
use std::collections::hash_map::DefaultHasher;
use std::hash::{Hash, Hasher};

/// 表示可存储在节点/边属性中的值类型
#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum Value {
    Null,
    Boolean(bool),
    Integer(i64),
    Float(f64),
    String(String),
    List(Vec<Value>),           // 支持嵌套列表

}
```

#### 2.4 Direction枚举（边的方向）
```rust
/// 遍历边时的方向
```

#### 2.5 Schema结构（模式定义）

### 3. Value类型实现细节

由于Value类型中包含f64，需要手动实现PartialEq、Eq和Hash trait以处理浮点数比较和哈希：

```rust
// 实现PartialEq以正确处理f64比较
impl PartialEq for Value {
    fn eq(&self, other: &Self) -> bool {
        match (self, other) {
            (Value::Null, Value::Null) => true,
            (Value::Boolean(a), Value::Boolean(b)) => a == b,
            (Value::Integer(a), Value::Integer(b)) => a == b,
            (Value::Float(a), Value::Float(b)) => (a == b) || (a.is_nan() && b.is_nan()), // 正确处理NaN
            (Value::String(a), Value::String(b)) => a == b,
            (Value::List(a), Value::List(b)) => a == b,
            (Value::Map(a), Value::Map(b)) => a == b,
            _ => false,
        }
    }
}

// 实现Eq trait，因为f64不实现Eq
impl Eq for Value {}

// 实现Hash trait以处理f64哈希
impl Hash for Value {
    fn hash<H: Hasher>(&self, state: &mut H) {
        match self {
            Value::Null => 0u8.hash(state),
            Value::Boolean(b) => b.hash(state),
            Value::Integer(i) => i.hash(state),
            Value::Float(f) => {
                // 从浮点数的位表示创建哈希
                if f.is_nan() {
                    // 所有NaN值应哈希到相同值
                    (0x7ff80000u32 as u64).hash(state);
                } else if *f == 0.0 {
                    // 确保+0.0和-0.0哈希到相同值
                    0.0_f64.to_bits().hash(state);
                } else {
                    f.to_bits().hash(state);
                }
            },
            Value::String(s) => s.hash(state),
            Value::List(l) => l.hash(state),
            Value::Map(m) => {
                // 按排序顺序哈希键值对
                let mut pairs: Vec<_> = m.iter().collect();
                pairs.sort_by_key(|&(k, _)| k);
                pairs.hash(state);
            },
        }
    }
}
```

当前的配置设计已经比较完善，将保持并优化：

```rust
use serde::{Deserialize, Serialize};

#[derive(Debug, Deserialize, Serialize, Clone)]
pub struct Config {
    pub host: String,              // 服务主机地址
    pub port: u16,                 // 服务端口号
    pub storage_path: String,      // 存储路径
    pub cache_size: usize,         // 缓存大小
    pub enable_cache: bool,        // 是否启用缓存
    pub max_connections: usize,    // 最大连接数
    pub transaction_timeout: u64,  // 事务超时时间（秒）
    pub log_level: String,         // 日志级别
}
```

实现配置的加载和保存功能：

```rust
impl Config {
    pub fn load<P: AsRef<Path>>(path: P) -> Result<Self, Box<dyn std::error::Error>> {
        let content = fs::read_to_string(path)?;
        let config: Config = toml::from_str(&content)?;
        Ok(config)
    }

    pub fn save<P: AsRef<Path>>(&self, path: P) -> Result<(), Box<dyn std::error::Error>> {
        let content = toml::to_string_pretty(self)?;
        fs::write(path, content)?;
        Ok(())
    }
}

impl Default for Config {
    fn default() -> Self {
        Self {
            host: "127.0.0.1".to_string(),
            port: 9758,
            storage_path: "data/graphdb".to_string(),
            cache_size: 1000,
            enable_cache: true,
            max_connections: 10,
            transaction_timeout: 30,
            log_level: "info".to_string(),
        }
    }
}
```

### 7. 实现说明

1. **数据结构映射**：将Nebula的C++数据结构转换为Rust，保留核心概念但使用Rust风格的实现

2. **内存安全**：利用Rust的所有权系统实现内存安全，无需手动内存管理

3. **性能优化**：使用HashMap和BTreeSet等高效数据结构

4. **序列化支持**：使用serde库实现数据的序列化和反序列化

5. **错误处理**：使用anyhow和thiserror实现优雅的错误处理

### 8. 迁移考虑

1. 与Nebula相比，graphDB的实现更简化，去除了分布式功能
2. 保留了核心的图数据模型（节点、边、属性）
3. Value类型支持基本数据类型和复合类型
4. 保持了模式定义功能
