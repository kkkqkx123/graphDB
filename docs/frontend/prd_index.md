## 使用指南

### 如何使用这些 PRD 文档

1. **阶段 1**: 从 [prd_phase1.md](./prd_phase1.md) 开始，搭建项目基础
2. **阶段 2**: 完成阶段 1 后，参考 [prd_phase2.md](./prd_phase2.md) 实现查询控制台
3. **阶段 3-5**: 依次参考各阶段的 PRD 文档实现 Schema 管理功能
4. **阶段 6**: 完成阶段 2 后，参考阶段 6 的 PRD 实现图可视化
5. **阶段 7**: 完成阶段 3 后，参考阶段 7 的 PRD 实现数据浏览

### PRD 文档结构

每个阶段的 PRD 文档包含以下章节：

1. **引言（Introduction）**: 阶段目标、核心价值、目标用户、范围
2. **功能需求**: 详细的功能描述、用户故事、验收标准
3. **非功能需求**: 性能、可用性、兼容性、安全、可靠性的要求
4. **用户界面需求**: 界面布局和组件的详细要求
5. **数据需求**: 数据持久化和验证的要求
6. **API 集成需求**: 需要调用的后端 API 端点
7. **测试需求**: 单元测试、集成测试、端到端测试的要求
8. **交付物**: 代码、文档、测试的具体交付清单
9. **验收标准**: 功能、质量、性能的验收标准
10. **风险和假设**: 项目风险和假设
11. **附录**: 术语表、参考文档、变更历史

## 注意事项

1. **渐进式开发**: 每个阶段都有明确的交付物和验收标准，可以独立交付和测试
2. **功能内聚**: 每个阶段的功能高度相关，减少上下文切换
3. **可独立交付**: 每个阶段完成后都可以独立测试和使用
4. **简化用户故事**: 纯数据操作功能整合为整体用户故事，避免过于细碎
5. **EARS 格式**: 所有验收标准都遵循 EARS（Easy Approach to Requirements Syntax）格式
