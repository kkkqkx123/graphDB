# GraphDB 前端功能清单

基于 nebula-studio-3.10.0 的功能分析，结合 GraphDB 项目的实际需求，整理出前端功能清单。

## 项目背景

- **GraphDB**: 基于 Rust 实现的轻量级单节点图数据库
- **目标**: 提供简单易用的 Web 管理界面
- **参考**: nebula-studio-3.10.0（作为功能参考，非完全照搬）

### 三、不需要实现的功能

#### 3.1 复杂数据导入功能 ❌

**原因**:
- GraphDB 是单节点本地数据库，主要用于个人和小规模应用
- 不需要支持大规模数据导入
- 可以通过 API 或脚本直接导入数据

**不实现的功能**:
- ❌ 远程数据源配置（S3、SFTP）
- ❌ 复杂的数据映射配置
- ❌ 导入任务管理和监控
- ❌ 导入日志查看
- ❌ AI 辅助导入

**替代方案**:
- 提供简单的 CSV/JSON 文件导入功能
- 提供 REST API 供外部系统导入数据
- 提供命令行工具进行数据导入

**参考模块**: `pages/Import/`（不参考）

---

#### 3.2 可视化建模工具（Sketch）❌

**原因**:
- 对于简单的单节点数据库，可视化建模不是必需的
- 可以通过 Schema 管理界面直接创建和修改 Schema
- 增加开发复杂度

**不实现的功能**:
- ❌ 拖拽式图结构设计
- ❌ Schema 草图保存
- ❌ 可视化建模界面

**替代方案**:
- 在 Schema 管理界面提供直观的表单操作
- 提供 DDL 语句的直接编辑功能

**参考模块**: `pages/SketchModeling/`（不参考）

---

#### 3.3 LLM 集成功能 ❌

**原因**:
- GraphDB 定位为轻量级数据库，不需要 AI 功能
- 增加外部依赖和复杂度
- 用户群体主要是开发者，不需要自然语言查询

**不实现的功能**:
- ❌ AI 聊天机器人
- ❌ 自然语言转 Cypher 查询
- ❌ AI 辅助数据导入

**参考模块**: `pages/LLMBot/`（不参考）

---

## 技术栈建议

### 前端框架
- React 18 + TypeScript
- Vite（构建工具）

### UI 组件库
- Ant Design 5.x

### 状态管理
- Zustand（轻量级）或 MobX（与 nebula-studio 一致）

### 代码编辑器
- Monaco Editor

### 图可视化
- Cytoscape.js 或 React Flow

### HTTP 客户端
- Axios

### 路由
- React Router v6

## 注意事项

1. **API 适配**: GraphDB 的 API 与 NebulaGraph 不同，需要根据实际 API 进行适配
2. **查询语言**: GraphDB 使用 Cypher，需要调整编辑器配置
3. **简化设计**: 专注于核心功能，避免过度设计
4. **渐进式开发**: 从基础功能开始，逐步添加高级特性
5. **性能考虑**: 单节点数据库的性能特点，优化前端体验
