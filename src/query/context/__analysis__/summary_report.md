# src/query/context 目录结构分析总结报告

## 执行摘要

本报告完成了对 `src/query/context` 目录的全面分析，识别了当前结构存在的问题，并提出了详细的重构方案。

## 分析结果概览

### 当前状态
- **文件总数**: 9个主要文件 + 6个验证子目录文件
- **总代码行数**: 约 5,000+ 行
- **最大文件**: `storage_expression_context.rs` (1600行)
- **最小文件**: `expression_eval_context.rs` (70行)

### 主要问题识别
1. **文件大小极度不均衡**: 3个文件超过800行，存在1600行的超大文件
2. **职责混杂**: 单个文件承担过多不同职责
3. **依赖关系复杂**: 存在潜在的循环依赖风险
4. **可维护性差**: 超大文件难以理解和修改

## 重构建议

### 核心重构方案
将当前扁平结构重构为层次化模块结构：

```
src/query/context/
├── ast/          # AST相关上下文
├── execution/    # 执行相关上下文
├── expression/   # 表达式相关上下文
├── request/      # 请求相关上下文
├── managers/     # 管理器接口
└── validate/     # 验证上下文（保持现有）
```

### 关键拆分操作

1. **`storage_expression_context.rs` (1600行)** → 拆分为4个文件
2. **`query_context.rs` (845行)** → 拆分为5个文件
3. **`request_context.rs` (938行)** → 拆分为4个文件
4. **`ast_context.rs` (383行)** → 拆分为多个查询类型文件
